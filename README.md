## 1. 사용자 설명서

### 1.1 설치 방법

#### 최신 버전 다운로드

👉 **[GitHub Releases 페이지](https://github.com/zaolee/jafm-electron/releases/latest)** 에서 최신 EXE 파일을 받으세요.

---

#### Windows (권장)

1. 위 링크에서 `자금일보-Setup-x.x.x.exe` 를 다운로드합니다.
2. 파일을 더블클릭하여 설치를 시작합니다.
3. 설치 위치를 선택하고 **설치** 버튼을 클릭합니다.
4. 설치 완료 후 바탕화면의 **자금일보** 아이콘을 더블클릭하여 실행합니다.

> ⚠️ Windows 보안 경고가 뜰 경우: "추가 정보" → "실행" 클릭

#### macOS

1. `자금일보-1.0.0.dmg` 파일을 다운로드합니다.
2. DMG 파일을 열고 **자금일보** 아이콘을 **응용 프로그램** 폴더로 드래그합니다.
3. 응용 프로그램 폴더에서 자금일보를 실행합니다.

---

### 1.2 처음 시작하기

프로그램을 처음 설치하면 반드시 **계좌 초기잔액**을 설정해야 합니다.

1. 오른쪽 탭에서 **⚙️ 설정** 클릭
2. 각 계좌의 **현재 실제 잔액**을 입력 (숫자만, 콤마 자동 입력됨)
3. **✅ 저장** 버튼 클릭

> 💡 이후 입출금을 입력하면 이 금액을 기준으로 잔액이 자동 계산됩니다.

### 1.3 탭별 기능 안내

#### 📋 자금 입력 탭

| 항목 | 설명 |
|------|------|
| 가용자금 합계 | 모든 원화 계좌 잔액의 합계 (상단 파란 박스) |
| 계좌별 잔액 카드 | 기업은행(주)/건설, 농협, 우리, 외화, 현금 |
| 입출금 내역 입력 | 날짜 · 계좌 · 입출금 구분 · 금액 · 거래처 입력 후 **저장** |
| 내역 목록 | 기간 필터로 조회, 삭제 가능 |

**입력 방법:**

```
날짜 선택 → 계좌 선택 → 입금/출금 선택 → 금액 입력 → 거래처 입력 → ✅ 저장
```

---

#### 📊 보고서 탭

1. **보고 기간** 시작일과 종료일 선택 (예: 월요일 ~ 일요일)
2. **🔍 보고서 생성** 클릭
3. 화면에 주간자금일지 양식으로 보고서 생성
4. **🖨️ 인쇄** 버튼으로 출력

보고서 포함 내용:
- 결재란 (담당 / 이사)
- 계좌별 잔액 현황 (전주잔액 / 금주입금 / 금주출금 / 현잔고)
- 입금 상세 내역
- 출금 상세 내역
- 가용자금 합계

---

#### 💾 내보내기 탭

**엑셀 다운로드:**
1. 기간 선택
2. **📥 엑셀 다운로드** → 저장 위치 선택
3. 파일에 **입출금내역**, **계좌잔액현황** 시트 2개 생성

**은행 거래내역 가져오기:** → [1.4 은행 파일 가져오기](#14-은행-파일-가져오기) 참고

**데이터 관리:**
- `💾 JSON 백업`: 전체 데이터를 파일로 저장
- `📂 JSON 복원`: 백업 파일에서 데이터 불러오기
- `🗑️ 전체 삭제`: 모든 내역 삭제 (2회 확인)

---

#### ⚙️ 설정 탭

각 계좌의 **기준 잔액** 설정.  
프로그램 첫 사용 또는 새 회계연도 시작 시 업데이트합니다.

### 1.4 은행 파일 가져오기

#### 은행별 엑셀 내보내기 방법

| 은행 | 경로 |
|------|------|
| 기업은행 | 인터넷뱅킹 → 조회/이체 → 거래내역조회 → 엑셀저장 |
| 농협은행 | 인터넷뱅킹 → 조회 → 거래내역조회 → 내보내기 → 엑셀 |
| 우리은행 | 인터넷뱅킹 → 조회 → 거래내역조회 → 파일저장(Excel) |
| 국민은행 | 인터넷뱅킹 → 조회 → 입출금내역 → 엑셀 다운로드 |
| 신한은행 | SOL뱅크 → 조회 → 거래내역조회 → 엑셀 저장 |

#### 가져오기 순서

1. **💾 내보내기** 탭 이동
2. **은행 선택** 드롭다운에서 해당 은행 선택
3. **계좌 구분** 선택 (어느 계좌로 등록할지)
4. 📋 **은행별 엑셀 컬럼 구조 보기** 버튼으로 파일 형식 미리 확인 가능
5. **📂 파일 선택** 클릭 → 은행에서 내려받은 xlsx 또는 csv 파일 선택
6. 미리보기 확인 → 불필요한 항목 체크 해제
7. **✅ 선택 항목 가져오기** 클릭

> ✅ 같은 날짜·내용·금액의 중복 항목은 자동으로 제외됩니다.

### 1.5 데이터 백업과 복원

#### 백업 권장 주기: **주 1회**

**백업 방법:**
- 상단 메뉴 → **파일** → **데이터 백업 (JSON)** (단축키: `Ctrl+S`)
- 또는 **💾 내보내기** 탭 → **JSON 백업**

**복원 방법:**
- **💾 내보내기** 탭 → **JSON 복원** → 백업 파일 선택
- 기존 데이터는 유지되고, 백업 파일의 데이터가 추가됩니다 (중복 제외)

#### 데이터 저장 위치

데이터는 PC의 다음 경로에 자동 저장됩니다:

```
Windows: C:\Users\사용자이름\AppData\Roaming\jafm\
macOS:   ~/Library/Application Support/jafm/
```

상단 메뉴 → **도움말** → **데이터 저장 위치 열기**로 바로 접근 가능합니다.

### 1.6 자주 묻는 질문

**Q. 프로그램을 삭제하면 데이터도 사라지나요?**  
A. 아닙니다. 데이터는 별도 폴더에 저장되므로 프로그램을 재설치해도 유지됩니다. 단, PC를 포맷하거나 AppData 폴더를 삭제하면 사라지므로 주기적으로 JSON 백업을 받아두세요.

**Q. 인터넷이 없어도 사용할 수 있나요?**  
A. 네. 완전히 오프라인으로 동작합니다.

**Q. 은행 파일을 올렸는데 인식이 안 돼요.**  
A. 은행 선택을 "자동 감지"에서 정확한 은행명으로 바꿔 다시 시도하세요. 그래도 안 되면 은행 앱 버전 업데이트로 컬럼명이 바뀐 경우일 수 있습니다. 개발자에게 해당 파일의 헤더 행을 공유해주세요.

**Q. 화면 글씨가 너무 작아요.**  
A. 상단 메뉴 → **보기** → **확대** (단축키: `Ctrl++`)로 화면을 키울 수 있습니다.

**Q. 계좌를 추가하거나 이름을 바꾸고 싶어요.**  
A. 현재 버전은 코드 수정이 필요합니다. `src/index.html`의 `ACCOUNTS` 배열을 수정하세요 (개발자 설명서 참고).

### 2.4 아키텍처 설명

```
┌─────────────────────────────────────────────────────┐
│                   Electron App                       │
│                                                     │
│  ┌─────────────────┐    IPC     ┌────────────────┐  │
│  │  Main Process   │◄──────────►│ Renderer Proc  │  │
│  │  (main.js)      │            │ (index.html)   │  │
│  │                 │  preload   │                │  │
│  │  - 파일 I/O     │◄──────────►│  - UI 렌더링   │  │
│  │  - 다이얼로그   │            │  - 비즈니스 로직│  │
│  │  - 메뉴 생성    │            │  - 데이터 처리  │  │
│  │  - 앱 생명주기  │            │                │  │
│  └────────┬────────┘            └────────────────┘  │
│           │                                         │
│           ▼                                         │
│  ┌─────────────────┐                                │
│  │  userData 폴더  │                                │
│  │  (로컬 저장)    │                                │
│  │                 │                                │
│  │  transactions   │                                │
│  │    .json        │                                │
│  │  init_balance   │                                │
│  │    .json        │                                │
│  └─────────────────┘                                │
└─────────────────────────────────────────────────────┘
```

#### 보안 원칙

- `contextIsolation: true` — renderer에서 Node.js API 직접 접근 불가
- `nodeIntegration: false` — XSS 공격 시 파일시스템 접근 차단
- `preload.js`의 `contextBridge`를 통해서만 IPC 통신 허용
- CSP(Content Security Policy) 메타태그로 외부 스크립트 로드 차단

### 2.5 주요 모듈 설명

|-----------|------|

| `ipcMain.handle('load-transactions')` | 거래 데이터 파일 읽기 |

| `ipcMain.handle('read-file')` | 바이너리 파일(xlsx/csv) 읽기 |

#### `src/preload.js`

`contextBridge.exposeInMainWorld('electronAPI', {...})`로 renderer에 노출하는 API:

| API | 설명 |
|-----|------|
| `saveTransactions(data)` | 거래 배열 저장 |
| `loadTransactions()` | 거래 배열 로드 |
| `saveInitBalance(data)` | 초기잔액 객체 저장 |
| `loadInitBalance()` | 초기잔액 객체 로드 |
| `saveFileDialog(opts)` | 저장 위치 선택 다이얼로그 |
| `openFileDialog(opts)` | 파일 선택 다이얼로그 |
| `writeFile(opts)` | 파일 쓰기 |
| `readFile(opts)` | 파일 읽기 |
| `onMenuBackup(cb)` | 메뉴 → 백업 이벤트 수신 |
| `onMenuRestore(cb)` | 메뉴 → 복원 이벤트 수신 |

#### `src/index.html` 주요 JS 함수

| 함수 | 설명 |
|------|------|
| `loadData() / saveData()` | 거래 데이터 로드/저장 (Electron/브라우저 자동 분기) |
| `loadInitBalance() / saveInitBalance()` | 초기잔액 로드/저장 |

| `addTransaction()` | 거래 입력 저장 |
| `renderReport()` | 주간자금일지 보고서 생성 |
| `doExport(list, filename)` | 엑셀 파일 생성 및 저장 |
| `importBankFile(event)` | 은행 파일 파싱 및 미리보기 |
| `confirmImport()` | 파싱 결과를 거래 데이터로 등록 |
| `renderSettingsForm()` | 초기잔액 설정 폼 렌더링 |

### 2.6 데이터 구조

#### transactions.json

```json
[
  {
    "id": 1709000000000,
    "date": "2026-03-03",
    "account": "기업은행(주계좌)",
    "type": "입금",
    "amount": 55000000,
    "desc": "2026 대한민국교육박람회 행사장 운영",
    "note": ""
  },
  {
    "id": 1709000000001,
    "date": "2026-03-04",
    "account": "기업은행(주계좌)",
    "type": "출금",
    "amount": 2671995,
    "desc": "BC카드 선결제",
    "note": ""
  }
]
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | number | 타임스탬프 기반 고유 ID |

### 2.7 배포 방법 (GitHub Actions 자동 빌드)

#### 동작 방식

```
소스 수정 → 커밋 → 버전 태그 push
      ↓
GitHub Actions 자동 실행 (약 5~10분)
      ↓
EXE 빌드 → GitHub Releases에 자동 업로드
      ↓
사용자가 앱 켜면 "업데이트 있음" 팝업 자동 표시
```

#### 배포 순서

**1. package.json 버전 변경**
```json
"version": "1.0.1"
```

**2. 커밋 (VSCode 소스 컨트롤 UI 또는 터미널)**
```bash
git add .
git commit -m "chore: v1.0.1 배포"
git push
```

**3. 태그 push (터미널 필수)**
```bash
git tag v1.0.1
git push --tags
```

**4. 빌드 확인**
- GitHub → Actions 탭에서 진행 상황 확인
- 완료 후 Releases 탭에 EXE 자동 업로드

### 2.8 커밋 컨벤션

#### 기본 형식

```
타입: 내용
```

#### 타입 종류

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새 기능 추가 | `feat: 대출관리 탭 추가` |
| `fix` | 버그 수정 | `fix: 잔액 계산 오류 수정` |
| `ui` | 화면/스타일 수정 | `ui: 거래내역 테이블 정렬 개선` |
| `refactor` | 코드 정리 (기능 변화 없음) | `refactor: transactions.js 분리` |
| `chore` | 설정, 빌드, 패키지 | `chore: electron-updater 추가` |
| `docs` | 문서, 주석 | `docs: README 업데이트` |

#### 버전 규칙 (Semantic Versioning)

```
v1.0.0
  │ │ └── 패치: 버그 수정
  │ └──── 마이너: 기능 추가
  └────── 메이저: 대규모 변경/전면 개편
```

| 상황 | 변경 예시 |
|------|---------|
| 버그 수정 | 1.0.0 → 1.0.1 |
| 기능 추가 | 1.0.0 → 1.1.0 |
| 전면 개편 | 1.0.0 → 2.0.0 |
